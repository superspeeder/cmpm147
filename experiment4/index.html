Experiment 4 - Infinite Worlds

World with Height

This is a bit awkward to view in the iframe, so you can also view it here: world_with_height/index.html

Artist Statement

Infinity I generated an infinite hilly landscape in an isometric view. I generate two distinct kinds of terrain: plains and hills.

Variation in Space The heightmap of the world is never the same anywhere, so you can find unique shapes in the terrain wherever you go.

Variation in Key The world key serves two purposes in this generator. The key is used to seed the noise function which generates the world, and the length of the key is used to determine the scaling of the noise used (longer keys mean that the world will be more stretched and have more gradual changes in terrain)

Signs of Life Anywhere you can go, you can adjust the height of tiles on the grid. These changes are kept between seeds (which can prove very interesting), and can only be removed by manually reverting the change (since you can both raise and lower terrain by clicking) or by using the reset button I added in to the webpage. Additionally (since it helps with clicking on tiles since I couldn't get it to figure out which tile is actually under the cursor accounting for height and perspective) the column that your cursor is hovering over is highlighted (though in this case i guess it's not exactly being lit up but made darker, just enough that it's visible).

Technical Details

This generator is a pretty simple noise generator which just renders each tile in a column (skipping the ones that might not be visible), moving up by a certain amount for each tile to make the height convincing. I generate different biomes by adjusting the added height and amplitude of the noise function I sample using a more stretched noise function put through a smoothstep function (letting me get nice edges to biomes instead of sharp edges).

Infinite Waterways

This is a bit awkward to view in the iframe, so you can also view it here: infinite_waterways/index.html

Artist Statement

Infinity I generate an infinite land of small rivers in some flat grass. The world has no elevation (other than the visible height change down to water level in the rivers, which helps sell them as rivers).

Variation in Space The actual rivers are not the same anywhere, and are based on noise.

Variation in Key The world key in this generator acts as the seed for noise and the length of it determines the concentration of rivers (longer keys get closer to everything being rivers).

Signs of Life The world is pretty bland only being rivers and grass, so I gave the user the ability to place trees to make things a bit more interesting. Anywhere you go, you can place a tree (I promise they will never be placed in the water).

Technical Details

This one is a bit interesting on the noise front, since I make the noise have ridges instead of just plain perlin noise (I mainly just check a threshold to see if its good). I did this in hopes that it would make the rivers look a bit more connected (which didn't work as well as I hoped, but I thought it worked well enough). I also do some autotiling to get the rivers to render nicely (letting me seperate the choice of river vs grass and the choice of which tile from my tileset to use).

Infinite Wordsearch

This is a bit awkward to view in the iframe, so you can also view it here: infinite_wordsearch/index.html

Artist Statement

Infinity I generated an infinite wordsearch for the word used as the key. I thought this to be an interesting style of infinite world since it's not so much a world as we normally think of, with terrain or cities, but it's a world of letters. Instead, this is infinite wordsearchland.

Variation in Space You can scroll forever in any direction and find completely different layouts of characters, meaning that the organization of correct words in the wordsearch should never be repeating.

Variation in Key The key is used to seed the noise function used to pick letters, but the actual letters and the correct word come from the world key itself.

Signs of Life All of the letters that you see can be clicked on to cycle them to other letters in the key, automatically updating the circling of correct words along with it. You have the power to make the correct words (which might be the only reasonable way to see some correct words with really long non-self-similar keys).

Technical Details

I did the selection of letters by simply sampling noise but at very different positions (I multiplied tile positions by 10 to make sure that I wasn't anywhere near the gradient of the original noise), scaling the output so that I could use it to index the world key, and then using the number to index the key. When you click on a tile, it just adds 1 to a counter which gets added to the index (and then the result of index modulus world key length is used as the actual index to make it cycle). Letters that are not the original value (the index offset modulus key length isn't 0) are colored blue as an indicator (makes it easier to see what is happening when you click something). I find correct words by searching each cardinal direction whenever a letter is the same as the start of the world key. If I can make the whole word, then I draw the circle around the word. Drawing the circle took some doing, mostly just making sure I do the right rotation to line it up with the isometric perspective.
